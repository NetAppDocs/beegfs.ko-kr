= BeeGFS CSI 드라이버 배포를 위한 전제 조건
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font

[role="lead"]
BeeGFS CSI 드라이버를 배포하고 구성하기 전에 실행 중인 Kubernetes 클러스터가 있는지, BeeGFS 클러스터에 대한 액세스 권한이 있는지, 클러스터 전체 리소스를 설치할 수 있는 적절한 권한이 있는지 확인하세요.



== Kubernetes 설정 확인

Kubernetes 클러스터의 Pod 상태를 확인하려면 다음 명령을 실행하세요.

[source, console]
----
kubectl get pods
----
**예상 출력:**

[listing]
----
root@node01:/etc/cni/net.d# kubectl get pods
No resources found in default namespace.
----
실행 중인 포드가 없으면 목록은 비어 있게 됩니다.  그렇지 않으면 포드는 오류 없이 나열되어야 합니다.



== BeeGFS 클라이언트 및 서비스 검증

다음 명령을 실행하여 BeeGFS 클러스터가 정상이고 모든 서비스에 접속할 수 있는지 확인하세요.

**비지에프에스 7.4**

[source, console]
----
beegfs-check-servers
----
**비지에프에스 8**

[source, console]
----
beegfs health check
----

NOTE: CSI 드라이버는 다음 두 가지와 함께 작동합니다. `beegfs-dkms-client` 그리고 `beegfs-client` 각 Kubernetes 노드에 설치해야 합니다.  설치 단계는 BeeGFS 클라이언트 DKMS 설명서를 참조하세요. https://doc.beegfs.io/latest/advanced_topics/client_dkms.html[]
